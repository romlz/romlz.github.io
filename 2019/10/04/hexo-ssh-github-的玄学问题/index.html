hexo ssh github 的玄学问题

发表于 2019-10-04

hexo d 的上传问题

在我把我的文章以及hexo的主题配置好了之后，在本地运行毫无问题，但是在使用hexo d指令的时候却遇到了令我无法解决的蜜汁BUG

我的hexo和github是使用ssh的，这样的好处是每次上传不需要用户密码验证，但是问题就出在这里

##网上解决方法参考

方法一（不可行）：

一开始，我一看是22端口问题，小意思，直接百度一下，把22端口开启了
但是开启了之后还是不行，依旧报错。
[百度结果](https://www.cnblogs.com/Diamond-sjh/p/11352156.html)

方法二（不可行）:

找到git/etc/ssh/ssh_config
增加以下内容
deploy:
  type: git
  repo: https://github.com/romlz/romlz.github.io.git
  branch: master
    弊端：
     在此后的每次hexo d的时候，需要手动输入github账户的账号密码，有点麻烦！！

[网络问题，用自己手机无线网作为热点即可](https://blog.csdn.net/zj57356498318/article/details/100513450)
在兜兜转转之后，我无意中发现这一篇文章，随即试了一下，还真可以了！！
但是还是不明白为何如此.... 真是玄学啊！！
